## Method

The script replays the saved R6c PPO policies from the Huawei result zips and
extracts the policy hidden state at every PPO policy call.

R6c uses K-window execution: one policy call can generate multiple internal
daily trading rows. For Stage 1, the hidden state from that policy call is
broadcast to the daily rows generated by that macro-step. This keeps the SSL
code aligned with the actual controller decision that produced the daily
behavior.

The public Joseph-compatible files are:

- `train_codes.parquet`: `date`, `code_id`, `valid`
- `code_summary.csv`: per-code return and market-context summary
- `stage1_metrics.csv`: K sweep with Joseph's Stage 1 metric schema

Additional R6c-specific files:

- `behavior_log_daily.parquet`: code assignments joined to daily R6c logs
- `code_behavior_summary.csv`: per-code cash/risk/confidence/top-k/group means
- `r6c_stage1_hidden_state_package.npz`: hidden states, action params, code ids,
  selected behavior arrays

## Current K=8 Result

The selected codebook uses `K=8` and a centered `17` trading-day window.

Important sanity check:

- replayed validation dates match the original R6c `validation_daily.csv`
- key metrics differ only at floating-point tolerance, around `1e-7`

This means hidden states and behavior logs are aligned with the true saved R6c
policy-forward behavior.

## Interpretation Link

The useful join key for Stage 2 is:

`date`

Recommended filter:

`valid == True`

The most useful behavior columns for SSL interpretation are:

- root/cash: `q_anchor`, `q_scheduled`, `q_target`, `cash_anchor`,
  `cash_scheduled`, `cash_target`
- confidence: `risk_stress`, `recovery_score`, `confidence_derisk`,
  `confidence_rerisk`
- event logic: `recovery_trigger`, `risk_break_trigger`,
  `window_closed_early`, `stop_active`
- trading intensity: `turnover_l1`, `stock_turnover_l1`, `trade_buy_count`,
  `trade_sell_count`, `trade_buy_weight_l1`, `trade_sell_weight_l1`
- Top-K: `incremental_topk_*`, `topk_*`
- group-aware layer: `incremental_topk_group_*`
- market context: `market_feature_*`

`code_behavior_summary.csv` is the first-pass map from SSL primitive code to
interpretable trading behavior.
